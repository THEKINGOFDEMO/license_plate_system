# Vue 前端说明

## 文档范围

这份文档只覆盖当前 Vue 前端页面与交互，包括：
- 登录页
- 首页
- 车牌识别页
- 历史记录页
- 系统说明页
- 前端启动与联调方式

这一步不包含模型训练或后端接口重构。

## 页面与路由

当前前端包含以下路由：
- `/login`
- `/`
- `/recognize`
- `/records`
- `/about`

未登录访问首页、识别页、历史记录页和系统说明页时，会自动跳转到 `/login`。

## 登录说明

当前登录为演示级实现：
- 默认账号：`admin`
- 默认密码：`123456`

登录成功后会在浏览器 `localStorage` 中保存登录状态。

退出登录后会：
- 清除登录状态
- 跳转回登录页

## 启动 Django 后端

启动前端前，请先确保 Django 后端已启动。

```bash
cd D:/django_project/laji/lunwen/license_plate_system
python backend/manage.py migrate
python backend/manage.py runserver
```

默认后端地址：

```text
http://127.0.0.1:8000
```

健康检查可访问：

```text
http://127.0.0.1:8000/api/health/
```

## 配置前端 API 地址

前端统一在：

```text
frontend/src/api/request.js
```

中配置 axios `baseURL`。

默认值：

```text
http://127.0.0.1:8000
```

也支持通过 Vite 环境变量覆盖：

```powershell
$env:VITE_API_BASE_URL="http://127.0.0.1:8000"
```

## 如何测试上传识别

1. 确保 Django 后端已启动
2. 登录系统
3. 进入“车牌识别”页面
4. 选择一张车辆图片
5. 点击“开始识别”

成功后应看到：
- 识别状态
- 车牌号
- YOLO 置信度
- bbox 坐标
- 标注结果图
- 车牌裁剪图

如果未检测到车牌，应看到：

```text
未检测到车牌，请尝试上传更清晰、车牌区域更明显的图片。
```

## 如何测试历史记录分页

1. 先完成若干次图片识别
2. 进入“历史记录”页面
3. 使用：
- 状态筛选
- 关键词搜索
- 每页条数切换

重点检查：

- 详情弹窗是否能打开

## 跨域说明

如果浏览器出现跨域错误，需要检查 Django 是否允许前端地址：

```text
http://127.0.0.1:5173
```

当前后端已经接入 `django-cors-headers`。如果仍有问题，检查：
- `CORS_ALLOWED_ORIGINS`
- `DJANGO_DEBUG`
- 后端是否重启

## 常见问题

### 1. 前端提示网络错误

原因：
- Django 后端未启动
- `VITE_API_BASE_URL` 配置错误

处理：
- 先访问 `http://127.0.0.1:8000/api/health/`
- 再检查前端环境变量

### 2. 图片不显示

原因可能是：
- Django 没有正确暴露 `media/`
- 后端返回的是本地路径而不是 URL
- 历史记录里是路径，需要前端转换成 `/media/...`

当前前端会优先使用：
- `annotated_image_url`
- `crop_image_url`

历史记录页也会尽量把路径转换成可访问的图片地址。

### 3. 登录后刷新页面

当前登录状态保存在 `localStorage`，刷新页面后仍会保留登录状态；如果手动清除浏览器存储，则需要重新登录。

至少满足这些条件：

5. 首页、识别页、历史记录页、系统说明页都能正常访问
6. 历史记录支持分页、筛选、搜索和详情查看
7. 页面文案、导航和布局整体完整稳定
